Class Hierarchy

  • optparse.OptionContainer: Abstract base class.
    • optparse.OptionParser: Class attributes: standard_option_list : [Option] list of standard options that will be accepted by all instances of this parser class (intended to be overridden by subclasses).
      • SBLsamkit.Parser.OptionParser: This class adds ARGUMENT_LIMIT options for a given option name.
  • ConfigParser.RawConfigParser
    • ConfigParser.ConfigParser
      • ConfigParser.SafeConfigParser
        • SBLsamkit.Parser.INIParser: An extended form of SafeConfigParser which stores the order of options for each section
  • object: The most base type
    • unreachable.Alignment: Alignment abstract base type:
      • unreachable.AlignmentWithSequenceReversal: Many aligners report the read's sequence in reverse-complemented form when it was mapped to the reverse strand.
        • unreachable.SAM_Alignment: When reading in a SAM file, objects of the class SAM_Alignment are returned.
          • SBLsamkit.SBLhtseq.SAM_Alignment: Subclass of the HTSeq.SAM_Alignment class.
    • exceptions.BaseException: Common base class for all exceptions
      • exceptions.Exception: Common base class for all non-exit exceptions.
        • SBLsamkit.Exceptions.ParserException: Base SBLsamkit exception which includes the stream name in the error message (the the stream is provided)
          • SBLsamkit.Exceptions.DuplicateStreamException: Custom exception for duplicate input stream names
          • SBLsamkit.Exceptions.IntRangeParserException: Custom exception specifically for NMrange,RPrange parser errors
          • SBLsamkit.Exceptions.LLParserException: Custom exception specifically for LL parser errors
          • SBLsamkit.Exceptions.OperatorParserException: Custom exception specifically for operator parser errors
          • SBLsamkit.Exceptions.OptionParserException: Custom exception for parser errors
          • SBLsamkit.Exceptions.OrderMissingFormatParserException: Custom exception for error dues to missing output format mode
          • SBLsamkit.Exceptions.OrderModeParserException: Custom exception specifically for order mode parser errors
          • SBLsamkit.Exceptions.PercentileParserException: Custom exception specifically for percentile argument parser errors
          • SBLsamkit.Exceptions.RnameListPairParserException: Custom exception specifically for RNAME pair list parser errors
          • SBLsamkit.Exceptions.RnameListParserException: Custom exception specifically for RNAME list parser errors
          • SBLsamkit.Exceptions.SSParserException: Custom exception specifically for SS parser errors
          • SBLsamkit.Exceptions.iLRangeParserException: Custom exception specifically for iLrange parser errors
    • SBLsamkit.Info.Entry: Define the basic ELN (Electronic Lab Notebook) entry list, which is a list of indented, text-wrapped strings bordered by a timestamp string
      • SBLsamkit.Info.RunEntry: Subclass of the basic Entry class which adds suport for the program run subsection of the .info file ELN entry
    • HTSeq.FileOrSequence: The construcutor takes one argument, which may either be a string, which is interpreted as a file name (possibly with path), or a connection, by which we mean a text file opened for reading, or any other object that can provide an iterator over strings (lines of the file).
      • SBLsamkit.SBLhtseq.SAM_Reader: Subclassed HTSeq.FileOrSequence which imitates what HTSeq.SAM_Reader does, except it yields our modified SAM_Alignment objects.
    • SBLsamkit.Info.InfoFile: Info file class, handles all aspects of the SBL .info spec 0.5.
    • SBLsamkit.MultiProcess.MPController: Main MP object which maintains various queue and task objects and info.
    • multiprocessing.process.Process: Process objects represent activity that is run in a separate process
      • SBLsamkit.MultiProcess.BasicWriter: Base writer class.
        • SBLsamkit.MultiProcess.Concatenator: Faux-writer class.
        • SBLsamkit.MultiProcess.SimpleWriter: Writer class for the output process.
          • SBLsamkit.MultiProcess.OrderedWriter: Writer class for the output process.
      • SBLsamkit.MultiProcess.TaskProcessor: Generic task processing object.
    • SBLsamkit.Utils.RecordCounter: This basic class tracks pairs read, and compares against total (or estimated total), to report a progress "heartbeat" to the console
    • SBLsamkit.SAM.SAMPair: A class describing a read pair.
    • SBLsamkit.Stream.SAMStream: Basic SAM Stream class describing common stream/filter operations.
    • SBLsamkit.MultiProcess.StreamProcessor: This is the processing guts object.
    • dict: dict() -> new empty dictionary.
      • SBLsamkit.Utils.OrderedDict: Custom dict subclass which stores each key that's added in an ordered list.
