Class StreamProcessor

source code

object --+
         |
        StreamProcessor

This is the processing guts object. It accepts a tuple of strings representing a record pair as read from the input file (by the MPController) and converts it in a SAMPair object. It uses the orderedStreams to process the SAMPair just as the SP process would, except that results are stored in a dict and include the task ID, stream name, output data, stats amd other accumultaed information. The output data will be pre- formatted so the OrderedWriter does not have to manipulate them.

The meat of the parallel-processing approach, a __call__ method which makes the task object callable. It converts the input tuple into a SAMPair object, runs it through each stream for processing in the normal fashion, and then passes the results to the output queue.

leftStream(self, stream, operator)

source code 

Return the substream declared to the left of a specific operator. So for instance if an operator sequence looks like "sam,order,LL,fastq" and we passed "order:LL" as the operator this should return "sam". It's a way of letting us apply an operator to a single output sub-stream. Note that if no sub-stream was declared to the left of the provided operator it's assumed to be "sam"
